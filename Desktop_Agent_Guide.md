# 🖥️ Desktop Agent 桌面端使用文档

> ✨ 本版本在原使用文档基础上加入了轻量图标，便于快速定位功能区域和操作流程。

## 📘 1. 文档说明

本文档基于 Desktop Agent 桌面端运行界面截图整理，主要说明桌面端各个关键区域的功能、常见操作流程、任务执行状态含义以及使用注意事项。

Desktop Agent 是一个本地桌面自动化助手。用户可以在输入框中输入自然语言任务，例如“打开浏览器，搜索无畏契约更新”，Agent 会控制鼠标和键盘，按照任务规划自动执行桌面操作。

## 📚 3. 左侧导航栏

左侧导航栏用于任务管理、历史任务检索和进入账号配置页面。

### 🏷️ 3.1 应用标题区

界面左上角显示：

```text
Desktop Agent
```

该区域表示当前运行的是 Desktop Agent 桌面客户端。

标题下方左侧有红、黄、绿三个圆点按钮，模拟桌面应用窗口控制区，常用于关闭、最小化、最大化窗口。

右侧的小箭头按钮用于折叠或收起侧边栏，方便扩大主工作区显示面积。

---

### 🔍 3.2 搜索历史任务

位置：左侧导航栏顶部，输入框占据一行。

界面文案：

```text
搜索历史任务
```

功能说明：

- 用于搜索已经执行过的历史任务。
- 可以输入任务关键词，例如：
  - 浏览器
  - Edge
  - QQ
  - 天气
  - 研究报告
- 系统会根据输入内容过滤左侧历史任务列表。
- 适合快速定位之前运行过的任务记录。

使用方式：

1. 点击“搜索历史任务”输入框。
2. 输入任务关键词。
3. 查看下方历史任务列表的匹配结果。
4. 点击目标历史任务，右侧主工作区会展示该任务详情。

---

### ➕ 3.3 新建任务按钮

位置：搜索框下方，蓝色按钮。

界面文案：

```text
+ 新建任务
```

功能说明：

- 用于创建一个新的桌面自动化任务。
- 点击后，主工作区会切换到新任务输入状态。
- 用户可以在底部输入框中输入新的自然语言任务。

典型使用场景：

```text
打开浏览器，搜索无畏契约更新
```

```text
打开 Edge 浏览器，搜索杨凌今日天气
```

```text
帮我在桌面上创建一个关键词文档
```

操作步骤：

1. 点击“+ 新建任务”。
2. 在主工作区底部输入任务描述。
3. 点击发送按钮。
4. 等待 Agent 执行任务。

### 🕘 3.4 历史任务列表

位置：左侧导航栏中部。

界面标题：

```text
历史任务
```

每条历史任务通常包含以下信息：

| 元素 | 说明 |
|---|---|
| 任务标题 | 展示任务的简短描述，过长时会省略 |
| 状态标签 | 显示任务是否完成、失败或运行中 |
| 步数 | 显示该任务执行了多少步 |
| 时间 | 显示任务创建或执行时间 |

截图中出现的状态包括：

| 状态 | 显示颜色 | 含义 |
|---|---|---|
| ✅ 已完成 | 绿色 | 任务已成功结束 |
| ❌ 失败 | 红色 | 任务执行失败或中途异常停止 |
| 🔵 运行中 | 蓝色 | 任务正在执行或尚未结束 |

示例任务项：

```text
打开浏览器，搜索无畏契约...
已完成  12步  17:43
```

```text
打开Edge浏览器搜索无畏...
失败  3步  17:41
```

```text
帮我在桌面上创建一个关...
运行中  4步  16:05
```

功能说明：

- 点击某个历史任务，可以查看该任务的详情。
- 任务列表可以帮助用户复盘 Agent 的执行过程。
- 失败任务可以作为调试依据，观察失败发生在哪一步。

主工作区用于展示当前任务的执行状态、任务规划、执行时间线以及任务输入框。

### 🟢 4.1 任务状态标签

位置：主工作区顶部左侧。

截图中显示：

```text
已完成
[历史记录]
```

功能说明：

| 标签 | 说明 |
|---|---|
| ✅ 已完成 | 表示当前选中的任务已经执行结束 |
| 历史记录 | 表示当前正在查看历史任务记录 |

如果任务正在运行，状态通常会显示为运行中；如果任务失败，则显示失败状态。

---

### 📝 4.2 当前任务标题卡片

位置：主工作区上方中央的大白色卡片。

截图中显示任务：

```text
打开浏览器，搜索无畏契约更新
```

下方状态信息：

```text
Step 12/50    当前：任务已完成
```

功能说明：

| 元素 | 说明 |
|---|---|
| 任务标题 | 当前任务的自然语言描述 |
| Step 12/50 | 当前执行到第 12 步，最大允许 50 步 |
| 当前：任务已完成 | 当前任务状态说明 |

字段含义：

- `Step 12/50` 表示该任务总共允许最多执行 50 步，目前已经执行到第 12 步。
- `当前：任务已完成` 表示 Agent 判断任务已经达成目标并停止执行。

使用价值：

- 用户可以快速判断任务是否完成。
- 可以知道 Agent 执行任务所用的步数。
- 如果任务在接近最大步数时仍未完成，可能说明任务描述不够明确或界面识别失败。

### 🧠 4.3 任务规划卡片

位置：任务标题卡片下方。

截图中显示：

```text
助手

任务规划

🌐 1. 打开浏览器应用程序
🔍 2. 在搜索框输入关键词
👀 3. 等待并查看搜索结果
📌 4. 提取并整理关键信息
```

功能说明：

任务规划卡片用于展示 Agent 对用户任务的理解和拆解结果。

每一项代表一个高层操作阶段：

| 规划步骤 | 功能说明 |
|---|---|
| 打开浏览器应用程序 | Agent 需要先启动或切换到浏览器 |
| 在搜索框输入关键词 | Agent 需要找到搜索框并输入搜索内容 |
| 等待并查看搜索结果 | Agent 需要等待页面加载，并识别搜索结果 |
| 提取并整理关键信息 | Agent 根据任务要求整理搜索结果或完成后续动作 |

使用价值：

- 帮助用户确认 Agent 是否理解了任务。
- 如果任务规划明显不符合预期，用户可以停止任务并重新输入更明确的指令。
- 可以作为任务执行过程的逻辑说明。

### ⏱️ 4.4 执行时间线

位置：任务规划卡片下方。

界面标题：

```text
执行时间线
```

每个步骤以卡片形式展示，例如：

```text
Step 1    点击目标元素    已完成    耗时 19.1s
Step 2    点击目标元素    已完成    耗时 11.7s
Step 3    点击目标元素    已完成    耗时 14.8s
```

功能说明：

执行时间线展示 Agent 实际执行的每一步动作。

每个步骤卡片包含：

| 元素 | 说明 |
|---|---|
| Step 编号 | 当前步骤序号 |
| 动作名称 | Agent 在该步骤执行的动作 |
| 执行状态 | 该步骤是否完成 |
| 耗时 | 该步骤从开始到完成所用时间 |

截图中的动作是：

```text
点击目标元素
```

说明 Agent 在这些步骤中主要执行了鼠标点击操作。

状态标签：

| 状态 | 含义 |
|---|---|
| ✅ 已完成 | 当前步骤执行成功 |
| ❌ 失败 | 当前步骤执行失败 |
| 🔵 运行中 | 当前步骤正在执行 |

使用价值：

- 查看 Agent 每一步做了什么。
- 发现任务失败时，可以定位失败步骤。
- 用于判断 Agent 是否重复点击、执行过慢或动作异常。
- 便于调试模型输出和坐标点击效果。

### ✍️ 4.5 任务输入框

位置：主工作区底部的大输入框。

界面提示：

```text
请输入任务，交给我来帮你完成
```

功能说明：

- 用户在这里输入新的自然语言任务。
- 支持输入中文任务描述。
- 输入后点击右侧发送按钮，Agent 开始执行任务。

任务描述建议：

```text
打开浏览器，搜索无畏契约最新更新，并总结前三条结果
```

```text
打开记事本，输入一段关于 GUI 自动化的说明
```

```text
打开 Edge 浏览器，搜索今天杨凌天气
```

更推荐的写法：

- 明确要打开哪个应用。
- 明确要搜索或输入什么内容。
- 明确最终目标是什么。
- 对敏感操作说明是否需要确认。

不推荐的写法：

```text
帮我弄一下
```

```text
找点东西
```

这类指令过于模糊，Agent 可能无法准确执行。

### 📤 4.6 发送按钮

位置：输入框右下角，圆形向上箭头按钮。

功能说明：

- 点击后提交当前输入框中的任务。
- 当输入框为空时，按钮通常呈浅色或不可用状态。
- 当输入有效任务后，按钮用于启动 Agent 执行。

操作步骤：

1. 在输入框输入任务。
2. 确认任务描述准确。
3. 点击发送按钮。

### 🚨 4.7 底部安全提示

位置：主工作区底部。

截图中显示：

```text
Agent 将控制你的鼠标和键盘 · 按 Esc 紧急停止
```

功能说明：

- 提醒用户 Agent 会直接控制本机鼠标和键盘。
- 如果 Agent 行为异常，可以按 `Esc` 紧急停止。
- 执行任务期间，用户尽量不要手动移动鼠标或切换窗口，否则可能干扰 Agent 判断。

安全建议：

- 执行任务前保存重要文件。
- 不要让 Agent 无人值守执行高风险操作。
- 涉及付款、删除、发送消息、提交表单等操作时，应要求人工确认。
- 如果发现鼠标移动异常，立即按 `Esc` 或点击停止按钮。

### 🔑 5.3 API Key 输入框

界面显示：

```text
API Key

显示

```

| 元素 | 说明 |
|---|---|
| API Key 输入框 | 用于填写模型服务密钥 |

### 🌐 5.4 Base URL 输入框

界面显示：

```text
https://dashscope.aliyuncs.com/compatible-mode/v1
```

功能说明：

Base URL 是模型服务接口地址。当前截图中使用的是阿里云 DashScope 的 OpenAI-compatible 接口地址。

用途：

- 告诉桌面端应该向哪个模型服务发送请求。
- 只要服务兼容 OpenAI Chat Completions，并支持视觉输入，就可以作为后端模型接口。

配置示例：

```text
https://dashscope.aliyuncs.com/compatible-mode/v1
```

---

### 🤖 5.5 Model Name 输入框

界面显示：

```text
qwen3-vl-plus
```

功能说明：

Model Name 指定使用的视觉语言模型名称。

当前配置说明：

- 使用的是 `qwen3-vl-plus`。
- 该模型需要支持图片输入。
- 该模型需要能够根据截图输出 GUI 操作指令。

注意事项：

- 模型名称必须和 Base URL 对应的服务端模型名称一致。
- 如果模型不支持视觉输入，Agent 无法根据截图判断下一步操作。
- 如果模型输出格式不稳定，可能导致任务失败。

---

### 🔁 5.6 Max Retry 输入框

界面显示：

```text
10
```

功能说明：

Max Retry 表示模型调用失败后的最大重试次数。

适用场景：

- 网络波动。
- 模型服务限流。
- 临时请求失败。
- 接口超时后重试。

建议：

- 普通使用可以设置为 3 到 10。
- 如果网络不稳定，可以适当提高。
- 如果频繁失败，应优先检查 API Key、Base URL 和模型名称，而不是一味提高重试次数。

---

### ⏳ 5.7 Timeout 输入框

界面显示：

```text
30
```

功能说明：

Timeout 表示单次模型请求的超时时间，单位是秒。

适用场景：

- 模型响应较慢时，超时时间可以适当提高。
- 如果希望失败更快返回，可以适当降低。

建议：

- 常规任务可设置为 30 秒。
- 复杂截图或模型响应慢时可设置为 60 秒。
- 如果经常超时，应检查网络或模型服务状态。

---

### 🧪 5.8 测试连接按钮

界面文案：

```text
测试连接
```

功能说明：

- 用于验证当前 API Key、Base URL、Model Name 是否可用。
- 点击后，桌面端会尝试连接模型服务。
- 如果连接成功，说明配置基本可用。
- 如果连接失败，需要检查配置项。

常见失败原因：

| 问题 | 可能原因 |
|---|---|
| API Key 无效 | 密钥错误、过期、权限不足 |
| Base URL 错误 | 地址填写错误或服务不可达 |
| 模型名称错误 | 服务端不存在该模型 |
| 网络异常 | 本机无法访问模型服务 |
| 权限不足 | 当前账号没有调用该模型的权限 |

---

### 💾 5.9 保存配置按钮

界面文案：

```text
保存配置
```

功能说明：

- 保存当前 API 配置到本地。
- API Key 会加密保存。
- 保存后，新任务会使用最新配置调用模型。

建议流程：

1. 填写 API Key。
2. 检查 Base URL。
3. 检查 Model Name。
4. 设置 Max Retry 和 Timeout。
5. 点击“测试连接”。
6. 测试成功后点击“保存配置”。

## 🏁 6. 任务状态说明

Desktop Agent 中常见任务状态如下：

| 状态 | 含义 | 用户操作建议 |
|---|---|---|
| 🔵 运行中 | Agent 正在执行任务 | 不要移动鼠标或切换窗口 |
| ✅ 已完成 | Agent 已完成任务 | 可以查看执行时间线和历史记录 |

## 🧭 7. 推荐使用流程

```text
打开 Desktop Agent
  ↓
点击左下角“账号与设置”
  ↓
填写 API Key
  ↓
填写 Base URL
  ↓
填写 Model Name
  ↓
点击“测试连接”
  ↓
点击“保存配置”
  ↓
点击“+ 新建任务”
```

1. 点击左侧“+ 新建任务”。
2. 在底部输入框输入任务。
3. 点击右下角发送按钮。
4. 等待 Agent 自动执行。
5. 查看任务状态卡片。
6. 查看执行时间线。

### 🕘 7.3 查看历史任务流程

1. 在左侧历史任务列表中找到目标任务。
2. 如果任务较多，可以使用“搜索历史任务”。
3. 点击目标历史任务。
4. 在右侧查看任务标题、状态、规划和执行时间线。
5. 根据步骤状态判断任务执行情况。

## ✨ 8. 任务指令编写建议

为了提高成功率，建议任务描述遵循以下原则。

### 🖥️ 8.1 明确应用

推荐：

```text
打开 Edge 浏览器，搜索无畏契约更新
```

不推荐：

```text
帮我搜一下
```

## ❓ 9. 常见问题与处理

### 🧯 9.1 任务失败怎么办？

可以按以下顺序排查：

1. 查看左侧历史任务中的失败记录。
2. 点击失败任务，查看执行时间线。
3. 判断失败发生在哪一步。
4. 检查任务描述是否明确。
5. 检查 API 配置是否正确。
6. 重新输入更具体的任务。

---

### 📍 9.2 Agent 点击位置不准确怎么办？

可能原因：

- 屏幕缩放比例过高。
- 多显示器坐标映射异常。
- 应用窗口位置发生变化。
- 截图尺寸与实际屏幕坐标不一致。
- 运行过程中用户移动了鼠标。

处理建议：

- 尽量在单显示器环境下使用。
- 将系统缩放调整为 100% 或推荐比例。
- 执行任务期间不要移动鼠标。
- 保持目标窗口在主屏幕。
- 重新执行任务。

---

### 🔍 9.3 搜索不到历史任务怎么办？

处理方式：

1. 清空搜索框关键词。
2. 输入更短的关键词。
3. 检查任务是否被删除。
4. 查看左侧列表是否滚动到对应位置。

---

### 🔧 9.4 API 配置后仍不能运行怎么办？

检查以下项目：

1. API Key 是否填写正确。
2. Base URL 是否可访问。
3. Model Name 是否存在。
4. 模型是否支持视觉输入。
5. 网络是否正常。
6. 是否点击了“保存配置”。
7. 是否通过了“测试连接”。

---

### 🚨 9.5 如何紧急停止？

如果 Agent 行为异常，可以：

1. 按键盘 `Esc` 紧急停止。
2. 点击界面中的停止按钮，如果当前版本提供。
3. 关闭 Desktop Agent 窗口。
4. 必要时在系统任务管理器中结束进程。

---

## 📌 10. 使用注意事项

1. Agent 会控制本机鼠标和键盘，运行时请勿随意移动鼠标。
2. 执行任务前建议保存重要文件。
3. 不建议无人值守运行高风险任务。
4. 不要把完整 API Key 截图或发送给他人。
5. 涉及付款、删除、发送消息等动作时应要求人工确认。
6. 如果任务描述复杂，建议拆分成多个小任务。
7. 如果连续失败，应先检查 API 配置和屏幕环境。
8. 运行任务时尽量保持目标窗口位于前台。
9. 尽量避免在任务执行中切换应用。
10. 遇到异常动作时立即按 `Esc` 停止。

## 🎉 12. 总结

Desktop Agent 桌面端主要围绕“任务创建、任务执行、历史记录、API 配置”四个核心功能展开。

用户的主要使用路径是：

```text
配置 API
  ↓
新建任务
  ↓
输入自然语言指令
  ↓
Agent 自动执行
  ↓
查看执行时间线
  ↓
通过历史任务复盘
```

从三张截图可以看出，当前桌面端已经具备较完整的任务执行闭环：

1. 左侧支持历史任务检索和任务管理。
2. 主工作区支持任务状态展示。
3. Agent 可以生成任务规划。
4. 执行时间线可以展示每一步动作和耗时。
5. 设置页面支持 API Key、Base URL、模型名称、重试次数和超时时间配置。
6. 底部提示明确说明 Agent 会控制鼠标键盘，并支持 `Esc` 紧急停止。

建议用户在使用时保持任务描述清晰，并在涉及敏感操作时增加人工确认要求，以获得更安全、稳定的自动化体验。
